# 블록체인 기반 덴탈보험 시스템
### Blockchain-Based Dental Insurance Platform

---

## 1. 프로젝트 개요

### 무엇을 만들었나?

> **보험 가입부터 청구, 지급, 만기환급, 대출까지**
> 보험의 전체 생애주기를 스마트 컨트랙트로 자동화한 시스템

기존 보험의 문제:
- 보험사가 모든 데이터를 독점 관리 → 투명성 없음
- 보험금 지급까지 수일~수주 소요
- 중간 처리 단계가 많아 비용 높음
- 서류 위조 및 허위 청구 가능성 존재

블록체인 보험이 해결하는 것:
- 모든 거래가 블록체인에 영구 기록 → 누구나 검증 가능
- 스마트 컨트랙트로 조건 충족 시 즉시 자동 지급
- 오라클이 병원 데이터를 검증 → 허위 청구 방지
- 코드가 곧 계약 → 보험사 임의 거절 불가

## 3. 핵심 기능 7가지

### ① 청약 심사 (Underwriting) — 스마트 컨트랙트 자동 심사

보험에 가입하려면 심사를 통과해야 합니다.
이 심사 로직이 **스마트 컨트랙트 코드 안에** 내장되어 있습니다.

```
자동 거절 조건
├── 18세 미만
├── 75세 초과
├── 보장한도 / 보험료 비율 100배 초과
└── 이미 활성 증권 3건 보유

통과 시 → 관리자 최종 승인/거절 → 증권 자동 발급
```

**강점:** 심사 기준이 코드로 공개 → 자의적 차별/거절 불가능

---

### ② 보험료 자동 수납 (Premium Auto-Collection)

매월 보험료를 피보험자가 직접 내야 하는 번거로움을 해결합니다.

```
피보험자: [자동납부 ON] 클릭 → approve 서명 (1회)
          ↓
Premium Scheduler (30초 주기 폴링)
          ↓
납입 기한 도래 + 잔액 충분 확인
          ↓
collectPremium() 자동 호출 → 보험료 자동 이체
```

**강점:** 가입자가 매번 서명 없이도 자동 처리, 연체 방지

---

### ③ 보험금 청구 및 오라클 자동 검증 (Oracle Verification)

보험금 청구 시 실제 병원 진료내역과 비교하여 자동 승인/거절합니다.

```
피보험자: 청구 제출 (치료코드 + 금액)
          ↓
ClaimSubmitted 이벤트 감지 (Oracle Service)
          ↓
병원 진료내역 조회 (HIRA API / Mock DB)
          ↓
├── VERIFIED        → 즉시 승인 + 보험금 자동 지급
├── AMOUNT_EXCEEDED → 금액 초과 거절
└── UNKNOWN_CODE    → 미등록 코드 거절
```

지원 치료 코드 23개 (실제 건강보험심사평가원 코드 기준):
- D0120 치과검진 / D2750 크라운 / D6010 임플란트 / D8080 교정 등

**강점:** 허위·과잉 청구 방지, 승인까지 수초 내 완료

---

### ④ HIRA API 연동 준비 (교체 가능한 Provider 패턴)

오라클 서비스는 **제공자 교체** 구조로 설계되었습니다.

```
.env 파일 설정만으로 전환 가능

HOSPITAL_PROVIDER=mock   → Mock DB 사용 (현재 테스트 중)
HOSPITAL_PROVIDER=hira   → 실제 건강보험심사평가원 API 연동
```

**강점:** 코드 변경 없이 실제 정부 API로 즉시 전환 가능

---

### ⑤ 만기환급금 자동 지급 (Maturity Refund)

보험 만기 시 납입 보험료의 일정 비율을 자동으로 환급합니다.

```
증권 생성 시 만기일 + 환급률(70%) 설정
          ↓
Maturity Watcher (10초 주기 폴링)
          ↓
만기일 도래 확인
          ↓
processMaturityRefund() 자동 호출
          ↓
총 납입액 × 70% → 피보험자 지갑으로 즉시 환급
```

예시: 총 납입 50 USDC × 70% = **35 USDC 자동 환급**

**강점:** 만기 처리 누락 불가, 계약 조건대로 즉시 이행

---

### ⑥ 약관대출 (Policy Loan)

납입 보험료를 담보로 즉시 대출받을 수 있습니다.

```
대출 한도 = 해지환급금(총납입 × 환급률%) × 80%
이자율    = 연 5% (관리자 변경 가능, 최대 30%)
이자 계산 = 원금 × 연5% × 경과일수 / 365
상환      = 원금 + 이자 일시 상환
```

**강점:** 보험을 해지하지 않고 유동성 확보 가능, 이자 계산이 블록체인에 기록되어 투명

---

### ⑦ 이중 통화 지원 (USDC + KRW)

동일한 보험 시스템이 달러와 원화 **두 가지 통화**로 동작합니다.

| 항목 | USDC (달러) | KRW (원화) |
|---|---|---|
| 토큰 | MockUSDC | MockKRW |
| 소수점 | 6자리 ($0.000001) | 0자리 (1원 = 1토큰) |
| 샘플 보험료 | $50/월 | ₩70,000/월 |
| 준비금 | $50,000 | ₩70,000,000 |
| 치료코드 한도 | D0120: $100 | D0120: ₩140,000 |

**강점:** 글로벌(USDC) + 국내(KRW) 동시 지원, UI 토글 하나로 전환

## 5. 스마트 컨트랙트 구조

### DentalInsurance.sol 주요 구조체

```solidity
struct Policy {        // 보험증권
    uint256 id;
    address patient;           // 피보험자 지갑 주소
    uint256 monthlyPremium;    // 월 보험료
    uint256 coverageLimit;     // 보장 한도
    uint256 totalPaid;         // 누적 납입액
    uint256 maturityDate;      // 만기일
    uint256 maturityRefundRate;// 만기환급률
}

struct Claim {         // 보험금 청구
    uint256 policyId;
    uint256 amount;
    string  treatmentCode;     // ADA 치료 코드
    ClaimStatus status;        // Pending/Approved/Rejected/Paid
    string  rejectReason;
}

struct Application {   // 청약 심사
    uint256 age;
    uint8   riskScore;         // 위험 점수 0~100 자동 산정
    ApplicationStatus status;  // Pending/Approved/Rejected
}

struct PolicyLoan {    // 약관대출
    uint256 loanAmount;
    uint256 borrowedAt;
    uint256 interestRate;      // 연 5%
    bool    active;
}
```

### 보안 설계

- **ReentrancyGuard**: 재진입 공격 방지 (이중 출금 취약점 차단)
- **Ownable**: 관리자 전용 함수 접근 제어
- **IERC20**: 표준 인터페이스만 사용 → 어떤 스테이블코인으로도 교체 가능
- **immutable stablecoin**: 배포 후 토큰 주소 변경 불가 → 사기 방지

## 6. 전체 보험 플로우

```
[가입 신청]
피보험자 → submitApplication() → 자동 심사 룰 통과 → Pending
관리자   → approveApplication() → 보험증권 자동 생성

[보험료 납부]
피보험자 → approve(보험료) → payPremium() → totalPaid 누적
또는
자동납부 ON → Premium Scheduler → collectPremium() 자동 호출

[보험금 청구]
피보험자 → submitClaim(치료코드, 금액) → ClaimSubmitted 이벤트
Oracle   → 진료내역 검증 → oracleVerifyAndProcess()
           VERIFIED → Claim 승인 + 즉시 지급

[만기환급]
Maturity Watcher → 만기 감지 → processMaturityRefund()
→ 총납입 × 환급률 → 피보험자 지갑으로 자동 이체

[약관대출]
피보험자 → requestPolicyLoan(policyId, amount) → 즉시 지급
피보험자 → repayPolicyLoan(policyId) → 원금 + 이자 상환
```

## 7. 강점 요약

### ① 완전한 자동화
사람의 개입 없이 청구 → 검증 → 지급이 수초 내 완료됩니다.
기존 보험: 수일~수주 / 이 시스템: **수초**

### ② 투명성
모든 거래(납입, 청구, 지급, 대출)가 블록체인에 영구 기록됩니다.
누구나 조회 가능 → **보험사의 임의 처리 불가**

### ③ 실제 API 연동 구조
Mock 데이터 → 실제 건강보험심사평가원 API로 환경변수 하나로 전환.
**프로덕션 배포 준비 완료**

### ④ 이중 통화
달러(USDC) + 원화(KRW) 동시 지원.
**글로벌 시장과 국내 시장 모두 대응** 가능

### ⑤ 보안
OpenZeppelin 검증된 라이브러리 사용.
재진입 공격 방지, 접근 제어, 불변 주소 설정.

### ⑥ 확장성
스테이블코인 인터페이스(IERC20)만 맞으면 어떤 통화로도 교체 가능.
치료 코드 DB, 심사 룰, 이자율 모두 관리자가 파라미터로 변경 가능.

## 8. 데모 시나리오 (발표 순서)

```
1. 서버 시작 확인
   → Hardhat Node / 배포 / UI / 오라클 / 워처 / 스케줄러

2. USDC 모드 — 전체 플로우
   → 청약 신청 → 관리자 승인 → 파우셋 → 보험료 납부
   → D0120 $80 청구 → 오라클 자동 승인 → 즉시 지급 확인

3. 오라클 거절 시나리오
   → D0120 $150 청구 → AMOUNT_EXCEEDED 거절
   → D9999 청구 → UNKNOWN_CODE 거절

4. KRW 모드 전환
   → 헤더 🇰🇷 버튼 → 원화 단위로 전체 UI 전환
   → ₩70,000 납부 → ₩140,000 청구 → 자동 지급

5. 약관대출
   → 대출 한도 조회 → 대출 신청 → 즉시 지급 → 이자 포함 상환

6. 만기환급 (시간 조작)
   → node scripts/advance-time.js 3600
   → 워처 자동 감지 → 환급금 자동 지급
```
